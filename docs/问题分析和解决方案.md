## 🔍 主要问题识别

通过分析 1540 个问题的评估结果，发现以下主要问题：

### 1. **记忆检索不准确** (114个错误，7.4%)
- **症状**: 回答"记忆中没有提到"或"没有在记忆中提及"
- **影响**: 直接导致答案错误

### 2. **日期/时间错误** (151个错误，9.8%)
- **症状**: 日期回答完全错误
- **示例**: "19 January, 2023" → "July 2023" ❌
- **影响**: 时间相关问题的准确率大幅下降

### 3. **格式不匹配** (109个错误，7.1%)
- **症状**: 语义正确但格式不对
- **影响**: BLEU/F1 分数低，但 LLM Judge 可能给分

## ⚠️ 关键发现：模型配置问题

### 问题 1: LLM Judge 使用了 DeepSeek

**当前配置** (`evaluation/metrics/llm_judge.py`):
```python
_MODEL = os.getenv("DEEPSEEK_MODEL", "deepseek-chat")  # 默认使用 DeepSeek
```

**问题**: 
- 论文中可能使用 GPT-4 作为 Judge
- DeepSeek 的评分标准可能与 GPT-4 不同
- 这可能导致评分偏差

### 问题 2: 回答生成可能使用了 DeepSeek

**当前配置** (`evaluation/src/memzero/search.py`):
```python
self.openai_client = OpenAI(
    api_key=os.getenv("DEEPSEEK_API_KEY", os.getenv("OPENAI_API_KEY")),
    base_url=os.getenv("OPENAI_BASE_URL", "https://api.deepseek.com"),
)
response = self.openai_client.chat.completions.create(
    model=os.getenv("MODEL"),  # 如果设置了 DEEPSEEK_API_KEY，会使用 DeepSeek
    ...
)
```

**问题**:
- 如果环境变量中设置了 `DEEPSEEK_API_KEY`，代码会优先使用 DeepSeek
- 论文中使用的是 `gpt-4o-mini`
- DeepSeek 的回答质量可能不如 GPT-4o-mini

## ✅ 解决方案

### 方案 1: 使用正确的模型（推荐）

**步骤 1**: 检查并修改 `.env` 文件

确保使用 OpenAI 的模型：
```bash
# .env 文件
MODEL="gpt-4o-mini"  # 使用 OpenAI 模型
OPENAI_API_KEY="your-openai-api-key"

# 不要设置 DEEPSEEK_API_KEY，或者注释掉
# DEEPSEEK_API_KEY="..."
```

**步骤 2**: 修改 LLM Judge 使用 GPT-4

修改 `evaluation/metrics/llm_judge.py`:
```python
# 改为使用 OpenAI 的模型
_MODEL = os.getenv("OPENAI_MODEL", "gpt-4o-mini")  # 或 "gpt-4"
_BASE_URL = "https://api.openai.com/v1"  # 使用 OpenAI API
```

**步骤 3**: 重新运行实验

```bash
# 1. 重新添加记忆（如果需要）
python run_experiments.py --technique_type mem0 --method add

# 2. 重新搜索和生成回答
python run_experiments.py --technique_type mem0 --method search --top_k 30 --output_folder results/

# 3. 重新评估
python evals.py --input_file results/mem0_results_top_30_filter_False_graph_False.json --output_file evaluation_metrics.json --max_workers 2
```

### 方案 2: 优化检索参数

如果模型配置正确，尝试优化检索参数：

**选项 A: 增加 top_k**
```bash
python run_experiments.py --technique_type mem0 --method search --top_k 50 --output_folder results/
```

**选项 B: 启用 filter_memories**
```bash
python run_experiments.py --technique_type mem0 --method search --top_k 30 --filter_memories --output_folder results/
```

**选项 C: 使用 Mem0+ (graph-based search)**
```bash
# 需要先重新添加记忆（启用 graph）
python run_experiments.py --technique_type mem0 --method add --is_graph

# 然后搜索
python run_experiments.py --technique_type mem0 --method search --top_k 30 --is_graph --output_folder results/
```

### 方案 3: 改进提示词

检查并优化 `evaluation/prompts.py` 中的 `ANSWER_PROMPT`，确保：
1. 明确要求回答格式（少于5-6个词）
2. 强调时间转换的重要性
3. 要求基于记忆回答，不要编造
